2. ANOMALY DETECTION MODEL (ENSEMBLE)
================================================================================

LOCATION: ensemble_model.pkl (pickled model)
CODE: outlier_ensemble_detector.py, smart_predictor.py

---WHAT IT DOES---
Detects abnormal power consumption by combining 3 algorithms:
  1. Isolation Forest (15.20% anomalies detected)
  2. SGD One-Class SVM (13.29% anomalies detected)
  3. Statistical Autoencoder/PCA (10.48% anomalies detected)
  ENSEMBLE VOTING: Requires 2/3 models to agree → Final: 5.20% anomalies

---USER INPUT (3 VALUES)---
File: smart_predictor.py
Function: SmartPredictor.predict()
Line: ~187

Input format:
  - Timestamp: "DD-MM-YYYY HH:MM" (e.g., "15-11-2025 15:30")
  - Server_ID: String (e.g., "S1", "S2", "S3")
  - Power_Usage_Watts: Float (e.g., 145)

---AUTO-CALCULATED FEATURES (6 TOTAL)---
File: smart_predictor.py
Function: SmartPredictor.predict()
Line: 128-152

1. Power_Usage_Watts → User provides (no calc)
2. hour: Extracted from timestamp (0-23)
3. day: Extracted from timestamp (1-31)
4. weekday: Extracted from timestamp (0=Monday, 6=Sunday)
5. Rolling_Mean: Mean of last 12 power readings (1 hour average)
6. Rolling_Std: Standard deviation of last 12 readings

Feature Extraction Code:
  dt = datetime.strptime(timestamp_str, "%d-%m-%Y %H:%M")
  hour = dt.hour
  day = dt.day
  weekday = dt.weekday()

---HISTORY HANDLING (KEY FEATURE)---
File: smart_predictor.py
Function: SmartPredictor.predict()
Line: 112-127

LOGIC:
  IF user provides >= 12 rows:
    Use last 12 rows from user data for rolling stats
  ELSE IF user provides < 12 rows:
    Fetch remaining records from original dataset
    Auto-complete to 12 records total
    
Code:
  if n_new_records >= self.window_size:
      history = server_data[['Power_Usage_Watts']].head(self.window_size)
  else:
      n_needed = self.window_size - n_new_records
      history = self.get_last_history_from_dataset(server_id, count=n_needed)

Example:
  User sends: 5 S1 readings
  Model needs: 12 readings for rolling_mean
  System: Use 5 from user + fetch 7 from original dataset

4. HISTORY HANDLING SYSTEM
================================================================================

---WHY HISTORY MATTERS---

File: smart_predictor.py, carbon_forecaster.py

Problem: 
  Single power reading = no context
  Can't calculate rolling mean without history
  Can't detect patterns

Solution:
  Maintain rolling window of past readings
  Use last 12 readings (1 hour) for context
  Auto-fetch from dataset if user doesn't provide

---ROLLING WINDOW MECHANISM---

File: smart_predictor.py
Function: SmartPredictor.predict()
Line: 112-127

Pseudocode:
  ```
  def process_new_reading(timestamp, server, power):
    if we_have_12_past_readings:
      use_these_12_readings
    else:
      fetch_missing_from_dataset
      combine_to_get_12
    
    rolling_mean = mean(12_readings)
    rolling_std = std(12_readings)
    
    predict(power, rolling_mean, rolling_std)
  ```

Example Flow:
  1. User sends: S1, 15:30, 145W
  2. System checks: Do we have S1's last 12 readings?
  3. YES? Use them
  4. NO? Fetch from dataset for S1
  5. Calculate rolling stats
  6. Predict

---SLIDING WINDOW FOR MULTIPLE READINGS---

File: smart_predictor.py
Function: SmartPredictor.predict()
Line: 142-155

When user sends multiple readings:
  Reading 1: 110W → Use old history
  Reading 2: 125W → Update window, drop oldest
  Reading 3: 95W → Update window again
  Reading 4: 150W → All using latest history

This captures temporal context!

---DATASET AUTO-FETCH---

File: smart_predictor.py
Function: SmartPredictor.get_last_history_from_dataset()
Line: 72-82

How it works:
  1. Load server_fin.csv (training data)
  2. Filter by Server_ID
  3. Get last N readings
  4. Use as context for new predictions

Code:
  server_data = self.original_data[
      self.original_data['Server_ID'] == server_id
  ]
  history = server_data[['Power_Usage_Watts']].tail(count)

File: carbon_forecaster.py
Function: CarbonEmissionForecaster.train_all_servers()

SARIMA uses ALL historical data:

  - Automatically learns:
    * Daily patterns

No manual sliding window needed.
